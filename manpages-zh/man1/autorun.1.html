NAME

autorun - 自动装载/卸载CDROMs并在装载后执行/path/to/cdrom/autorun  

总览

autorun [-lmqv?V] [-a EXEC] [-c CDPLAYER] [-e STRING] [-i MILLISEC] [-n STRING] [-t STRING] [--autorun=EXEC] [--cdplayer=CDPLAYER] [--interval=MILLISEC] [--lock] [--mountonly] [--notify=STRING] [--notify-insert=STRING] [--notify-eject=STRING] [--quiet] [--verbose] [--help] [--usage] [--version] [cdromdevices...]  

描述

autorun自动识别系统上所有可获得的CDROMs，在插入CD时载装它 们，并执行CD上可能包含的'autorun'可执行文件。要让普通用户 使用这样的功能，你必须在/etc/fstab中想要使用的CDROMs条目 上加入选项user,exec。

你也可以在KDE中使用Autorun.kdelnk。只要把它放到你的 Autostart文件夹中即可。

程序分析命令字串并用装载点路径来替换%P%参数。用设备路径来 替换%D%。然后就用/bin/sh -c "command string"来执行命令字 串；

 

选项

-a, --autorun=EXEC
.IP • .IP n(el. 如果换CD盘的话执行EXEC。该可执行文件必须位于CD盘上。例如， 如果你指定"--autorun=myprogram"，而你的CD盘装载到了 /mnt/cdrom下的话，在换盘后，autorun会尝试运行 "/mnt/cdrom/myprogram"。 (缺省为"/autorun")
-c, --cdplayer=CDPLAYER
.IP • .IP n(el. 如果插入的是唱片的话，运行CDPLAYER (缺省为"/usr/bin/kscd")
-i, --interval=MILLISEC
.IP • .IP n(el. 每次检查之间间隔MILLISEC (缺省为1000000)
-l, --lock
.IP • .IP n(el. 锁定装载好的介质(手工卸载)
-m, --mountonly
.IP • .IP n(el. 只进行装载/卸载(不执行任何内容)
-n, --notify=STRING
.IP • .IP n(el. 如果换盘则执行命令STRING (缺省为"/usr/bin/kfmclient refreshDesktop")
-t, --notify-insert=STRING
.IP • .IP n(el. 如果插入盘片则执行命令STRING，就不执行"/autorun"以及用 --autorun指定了的内容 (缺省为"/usr/bin/kfmclient openURL %P%")
-e, --notify-eject=STRING
.IP • .IP n(el. Command STRING that will executed if a CD was ejected 如果取出盘片则执行命令STRING (缺省为"")
-q, --quiet
.IP • .IP n(el. 不产生任何输出
-v, --verbose
.IP • .IP n(el. 产生详细的输出
-?, --help,
.IP • .IP n(el. 列出此帮助信息
--usage
.IP • .IP n(el. 列出一份简要用法信息
-V, --version
.IP • .IP n(el. 列出程序版本

长选项所有的强制或可选参数也适合任何相应的短选项。
