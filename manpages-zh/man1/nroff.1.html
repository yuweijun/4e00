NAME

nroff - 用 groff 模拟 nroff 命令  

总览 SYNOPSIS


nroff [ -h ] [ -i ] [ -mname ] [ -nnum ] [ -olist ] [ -p ] [ -rcn ] [ -S ] [ -t ] [ -Tname ] [ -U ] [ -v ] [ file ... ]
 

描述 DESCRIPTION

nroff shell 脚本用 groff 来模拟 nroff 命令。 -T 选项允许的有效参数是 ascii, ascii8, latin1, utf8, nippon, 以及 cp1047 。如果参数无效，或者没有给出 -T 选项， nroff 检测当前语言环境来选择一个默认的输出设备。它首先使用 locale 程序，然后检测环境变量 LC_ALL, LC_CTYPE, 和 LANG, 最后是 LESSCHARSET 环境变量。

-h 和 -c 选项与 grotty 的 -h (在输出中使用tab) 和 -c (使用老式的输出方案而不是 SGR 转义序列) 选项等价。 -C, -i, -n, -m, -o, 和 -r 选项的效果在 troff(1) 中详述。另外， nroff 忽略 -e, -q, 和 -s (在 troff 中没有实现) 选项，没有任何提示。 -p (pic), -t (tbl), -S (safer), 和 -U (unsafe) 选项被传给 groff. -v 显示版本号。  

环境 ENVIRONMENT

GROFF_BIN_PATH
一个冒号分隔的路径列表，在搜索 PATH 之前从中搜索 groff 可执行文件。如果没有设置，将使用 `/usr/bin'.
 

注意 NOTES

这个 shell 脚本一般用于配合 man(1) 使用，所以不输出警告信息。nroff 样式的字符定义 (在文件 tty-char.tmac 中) 会被读取，来模拟不可显示的符号。  

参见 SEE ALSO

groff(1), troff(1), grotty(1)
