NAME

tkerror - 为处理后台错误而调用的命令  

总览 SYNOPSIS

tkerror message

 

描述 DESCRIPTION

注意: 在 Tk 4.1 中把 tkerror 命令重新命名为 bgerror，原因是事件循环(这个命令通常被它所调用)现在是Tcl 的一部分了。为了反向兼容，如果存在 tkerror(或一个可自动装载的 tkerror)，则当前的 Tk 版本提供的 bgerror 仍然尝试调用它，所以定义了错误处理器的老脚本仍然可以工作，但你应该设法修改你脚本来使用 bgerror 替代 tkerror，因为对老名字的支持在不久的将来可能不复存在。如果调用它失败，bgerror 将贴出(post)一个对话框显示这个错误并向用户提供栈跟踪。如果你想拥有自己的错误管理，你应该直接使用 bgerror 而不是 tkerror。bgerror 的文档可在 Tcl 的文档中获得。

 

关键字 KEYWORDS

background error, reporting
