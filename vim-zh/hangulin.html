Vim documentation: hangulin

*hangulin.txt* For Vim version 8.0. 最近更新: 2017年7月 VIM 参考手册 By Chi-Deok Hwang and Sung-Hyun Nam 译者: Willis http://vimcdoc.sf.net 简介 *hangul*
我们的目的是能够在 Vim 的 GUI 版本中输入 Hangul，韩国语的一种书写方式。如果你 有 XIM 程序，那么你可以用另外一个 |+xim| 特性。基本上，这个程序是给那些没有 XIM 程序的人用的。 编译
下面用的是些简单的选项。你可以加上其它配置选项。 ./configure --with-x --enable-multibyte --enable-hangulinput \ --disable-xim 你现在应该查看 feature.h。如果 |+hangul_input| 特性在配置时已经加入，你可以设 置更多的选项: 如把键盘类型设为 2 bulsik 或者 3 bulsik。在这里，你可以找到类似 于下述例子中的关键字。 #define HANGUL_DEFAULT_KEYBOARD 2 #define ESC_CHG_TO_ENG_MODE /* #define X_LOCALE */ 环境变量
你应该设置 LANG 环境变量为韩国 locale，比如 ko、ko_KR、eucKR 或 ko_KR.UTF-8。 如果你设置了 LC_ALL 环境变量，它应该也被设成韩国 locale。 Vim 资源
你可能要设置 'encoding' 和 'fileencodings'。 下面有些例子: :set encoding=euc-kr :set encoding=utf-8 :set fileencodings=ucs-bom,utf-8,cp949,euc-kr,latin1 键盘
你可以通过修改 VIM_KEYWORD 或是 HANGUL_KEYBOARD_TYPE 环境变量改变你的键盘类型 (2 bulsik 或是 3 bulsik)。在 sh 里，只要这么做 (2 bulsik): export VIM_KEYBOARD="2" or export HANGUL_KEYBOARD_TYPE="2" 如果两者都设置了，VIM_KEYBOARD 有优先权。 Hangul 字体
如果用 GTK 版本的 gvim，应该设置 'guifont' 和 'guifontwide'。 例如: set guifont=Courier\ 12 set guifontwide=NanumGothicCoding\ 12 如果用 Motif 或 Athena 版本的 gvim，应该在 vimrc 中设置 'guifontset'。可以 在 .Xdefaults 文件里设置字体集。 $HOME/.gvimrc: set guifontset=english_font,hangul_font $HOME/.Xdefaults: Vim.font: english_font ! 以下是 Athena 的 hangul 菜单 *international: True Vim*fontSet: english_font,hangul_font ! 以下是 Motif 的 hangul 菜单 *international: True Vim*fontList: english_font;hangul_font: 要 注意 其中用的是 , (逗号) 还是 ; (分号)。 还有，不要设 ':set guifont'。如果这项存在，那么 gvim 忽略 ':set guifontset'。 这就意味着 Vim 不会提供字体集的支持，从而只有英语会正确显示，Hangul 就不会了。 在 "fontset" 特性被打开后，Vim 不再允许设置语法高亮的 "font" 为纯英语字体。 比如，如果你在 .gvimrc 里用 :set guifontset=eng_font,your_font 那么你的语法高亮应该这样写 :hi Comment guifg=Cyan font=another_eng_font,another_your_font 如果你只是这么写 :hi Comment font=another_eng_font 那么你会看到错误信息。要 注意 ！ hangul 字体 hangul_font 的宽度应该是英语字体 english_font 的宽度的两倍。 尚未支持的特性
不支持 Johab 字体。 不支持 Hanja 字体。 我们没有这方面的计划。 如果你确实想要这些特性，可以用控制台版本的 Vim，配上支持的终端模拟。 漏洞或者意见
